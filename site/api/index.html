API Reference

Our REST interface powers quantum-inspired legal analytics. Each response packages citation support, jurisprudential metadata, and interpretive ambiguity scores so downstream workflows stay audit-ready.

Authentication

Requests authenticate with a bearer token generated from your ActionsGPT workspace. Include the token in the Authorization header for every call. Tokens scope to organization data partitions for confidentiality. 

Authorization: Bearer <workspace-token>

Endpoints

Endpoint Description Key Inputs
POST /sources/search Semantic + citation aware retrieval for authorities. query, jurisdiction, max_results
POST /arguments/build Structured drafting with issue framing and supporting precedent. issue, facts, stance
POST /salience/score Measure argument robustness against jurisprudential lean scenarios. argument_id, bias_threshold

Response Anatomy

Each response includes citations, normalized reporter data, quantum-inspired ambiguity metrics, and a jurisprudential leaning vector. Use the metadata to visualize interpretive tensions or feed model risk dashboards. 

{
  "citation": "410 U.S. 113",
  "holding": "Defined privacy protections under the Fourteenth Amendment",
  "probabilities": {
    "textualist": 0.41,
    "purposivist": 0.68
  },
  "qubitPhase": 0.19,
  "ambiguity": 0.24
}
